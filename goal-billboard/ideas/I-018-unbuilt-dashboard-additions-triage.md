# I-018 — Triage the ~12 unbuilt dashboard additions (the evaporation example)

**What it is.** When the user asked *"do you have any more ideas of useful additions to our system
dashboard"* (23f93bfd, 2026-05-26), the assistant offered a 15-item menu — but only **3** were pursued (BG
ghost watcher, pinned next-step, decision-handoff drawer). The other **~12** were enumerated in a chat reply
and never tracked. This idea is to **triage those 12 into the `ideas/` folder under G18** so each is a
visible, tracked candidate rather than text lost in a transcript.

**Why it matters.** This is itself a textbook case of the evaporation class this whole floated-ideas audit
exists to catch — a dozen concrete dashboard ideas offered, three built, the rest gone. It's idea-capture,
not a build: the value is getting the 12 onto the board so the user can pick which (if any) graduate.

**Concrete first step.** Recover the 15-item dashboard-additions menu from session 23f93bfd, subtract the 3
already-built (BG ghost watcher, pinned next-step, decision-handoff drawer), and file each of the remaining
~12 as its own short `I-*` idea under G18 (or, if several are tiny, a single grouped idea with a checklist —
decide at triage time).

**Leverage: LOW** (idea-capture, not build — but exactly the evaporation class this audit targets). Awaiting
greenlight (surface-only by contract).
