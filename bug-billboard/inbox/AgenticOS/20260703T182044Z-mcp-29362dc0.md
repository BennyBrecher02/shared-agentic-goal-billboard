## Description
Adversarial sandbox audit 2026-07-03 (context/markdowns/audits/deploy-kit-init-edge-cases-2026-07-03.md). VERIFIED clobber paths in portable-kit/deploy-kit.sh when the target repo ALREADY has same-named files: (1) deploy_device_testing (L323-332) unconditionally `cp`s scripts/ios-simulator-capture.sh + scripts/android-emulator-capture.sh over any existing file — no backup, no .kit-proposed sibling, logged as a plain "+ device-wrapper" success line; a hand-tuned wrapper is silently reverted on EVERY re-run (verified: sentinel + post-run-1 hand edit both lost). Fixes: no-clobber/.kit-proposed posture (or a [KIT-PORTABLE] marker-line ownership check) for the wrappers+rules+gates; escape & and \ in sed replacements; fix the printed env-var name; don't abort init on billboard opt-out.
