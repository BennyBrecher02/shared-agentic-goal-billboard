# Device Wall: cold-cell mode-glyph (👻/🪟) dimmed to 0.4 is illegible over rain

## Description
wall.css:301 `.cell.st-cold .mode-glyph { opacity: .4 }` drops the per-device boot-mode silhouette to 2.19:1 worst-case (vs 5.04:1 for the same glyph at the live-cell 0.9 opacity). The glyph is the ONLY per-cell readout of whether that device will boot headless vs windowed; on a COLD cell (the exact state where you're about to boot it and most need that info) it's the least legible. Same opacity-defeats-plate class as the disabled-button bug. Fix: raise cold mode-glyph to ~opacity .7 (the cell-head plate + halo already carry it). Verified live via computed-style + contrast math.
