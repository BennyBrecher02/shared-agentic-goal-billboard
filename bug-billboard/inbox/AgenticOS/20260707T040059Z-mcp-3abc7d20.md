# Device Wall: disabled deck buttons illegible over rain (opacity 0.35 defeats the plate)

## Description
wall.css:481 `.deck-btn:disabled { opacity: 0.35 }` multiplies the whole button — glyph AND plate together — so the readability pass's per-element --plate backing no longer protects it. Disabled ARRANGE (the .unavail:disabled state that fires whenever 0 windowed devices exist — a normal, common state since headless is the default) measures 2.52:1 vs the deck plate best-case and 2.09:1 with a bright rain-head bleeding through the 0.72 plate. Below WCAG AA (3:1 for large text). This is the ONE functional element the legibility pass missed. Fix: raise disabled floor to opacity ~0.55, OR dim only the border/glyph via color (not the whole plate) e.g. `.deck-btn:disabled{opacity:1;color:var(--ink-mid);border-color:var(--ink-dim)}` so the plate stays fully opaque. Verified live via Playwright computed-style + contrast math at 1440.
